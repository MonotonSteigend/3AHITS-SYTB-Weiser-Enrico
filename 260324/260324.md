# Shell Script
### Einführung in Shell Scripts


```sh
#Hallo Welt

#Shell Script = Textdatei mit shell Befehlen
#Befehle ergeben das gleiche Ergebnis wie in der Shell ausgefuehrt
MYVAR=HALLO
echo $MYVAR WELT
echo Guten Morgen


FILE_NAME=TEST.txt
DIR_NAME=mydir
#mkdir "$DIR_NAME"

echo "Irgendein Text" > "$DIR_NAME/$FILE_NAME"


#double quotes ""
NAME="Enrico"
TEXT="HALLO $NAME"

echo $TEXT

#single quotes ''
TEXT2='HALLO $NAME'
echo $TEXT2

#curly braces {}
FILE=myname
#echo "$FILE_001.txt"
#echo "$FILE_002.txt"

echo "${FILE}_001.txt"
echo "${FILE}_002.txt"
```
**Shell Script sind mehere shell Befehle nacheinander in einer Textdatei.**

Der Interpreter des Shell Scripts kann am anfang der Datei mit einem shebang (#!) definiert werden.

Variablen werden im Shell Script mit ``$Variablen_name=Wert`` definiert, dabei muss auf Leerzeichen geachtet werden weil diese ohne angührungsstriche einen neuen Befehl signalisieren.``$Variablen_name=Wert 1`` würde nicht gehen, man müsste ``$Variablen_name="Wert 1"`` schreiben. 

In dem Beispiel: 
```sh 
FILE_NAME=TEST.txt
DIR_NAME=mydir
mkdir "$DIR_NAME"

echo "Irgendein Text" > "$DIR_NAME/$FILE_NAME"
```
muss man darauf Achten ``""`` um die beiden Variablennamen zu setzen um namensauflösung zu erzielen, mit ``''`` der String wirklich so an echo übergeben werden wie er da steht. (Ebenso bei mkdir)

In dem Beispiel: 
```sh 
#curly braces {}
FILE=myname
#echo "$FILE_001.txt"
#echo "$FILE_002.txt"

echo "${FILE}_001.txt"
echo "${FILE}_002.txt"
```
wird versucht einen Standard Prefix für Filenamen zu setzeun und basierend auf diesem weitere Filenamen zu erstellen. Hier muss man ``${FILE}_andere_text`` benutzen um den anderen Text vom Variablennamen zu trennen.
