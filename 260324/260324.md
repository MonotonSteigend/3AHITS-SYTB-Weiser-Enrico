# Shell Script
### Einführung in Shell Scripts


```sh
#Hallo Welt

#Shell Script = Textdatei mit shell Befehlen
#Befehle ergeben das gleiche Ergebnis wie in der Shell ausgefuehrt
MYVAR=HALLO
echo $MYVAR WELT
echo Guten Morgen


FILE_NAME=TEST.txt
DIR_NAME=mydir
#mkdir "$DIR_NAME"

echo "Irgendein Text" > "$DIR_NAME/$FILE_NAME"


#double quotes ""
NAME="Enrico"
TEXT="HALLO $NAME"

echo $TEXT

#single quotes ''
TEXT2='HALLO $NAME'
echo $TEXT2

#curly braces {}
FILE=myname
#echo "$FILE_001.txt"
#echo "$FILE_002.txt"

echo "${FILE}_001.txt"
echo "${FILE}_002.txt"
```
**Shell Script sind mehere shell Befehle nacheinander in einer Textdatei.**

Der Interpreter des Shell Scripts kann am anfang der Datei mit einem shebang (#!) definiert werden.

Variablen werden im Shell Script mit ``$Variablen_name=Wert`` definiert, dabei muss auf Leerzeichen geachtet werden weil diese ohne angührungsstriche einen neuen Befehl signalisieren.``$Variablen_name=Wert 1`` würde nicht gehen, man müsste ``$Variablen_name="Wert 1"`` schreiben. 

In dem Beispiel: 
```sh 
FILE_NAME=TEST.txt
DIR_NAME=mydir
mkdir "$DIR_NAME"

echo "Irgendein Text" > "$DIR_NAME/$FILE_NAME"
```
muss man darauf Achten ``""`` um die beiden Variablennamen zu setzen um namensauflösung zu erzielen, mit ``''`` der String wirklich so an echo übergeben werden wie er da steht. (Ebenso bei mkdir)

In dem Beispiel: 
```sh 
#curly braces {}
FILE=myname
#echo "$FILE_001.txt"
#echo "$FILE_002.txt"

echo "${FILE}_001.txt"
echo "${FILE}_002.txt"
```
wird versucht einen Standard Prefix für Filenamen zu setzeun und basierend auf diesem weitere Filenamen zu erstellen. Hier muss man ``${FILE}_andere_text`` benutzen um den anderen Text vom Variablennamen zu trennen. 

### Übung (Variablen)

Löse diese Aufgabe ohne ein shell-script zu schreiben.

Lege 2 Variablen für Directories an, eine Variable für den Filenamen und eine Variable für die Dateiendung (Extension).

Verwende nur die Variablen anstatt der Directory und Filenamen um folgendes zu lösen:

- Erzeuge die Directories (das zweite befindet sich in dem ersten).
- Lege in dem zweiten Directory eine Text-Datei mit Inhalt an, verwende Filenamen - und Extension (getrennt mit .).
- Gib den Inhalt der Datei (unter Verwendung der Variablen) aus.


```sh 
DIR1="dir1"
DIR2="dir2"
NAME="test"
FILE_EXTENSIONS=".txt"

mkdir -p "${DIR1}/${DIR2}/"
echo "Irgendein Text" > "${DIR1}/${DIR2}/${NAME}.${FILE_EXTENSIONS}"
```

Hier müssen Variablen in dern  geschwungenen Klammern und "" verwennndet werdn um die Namensauflösung korrekt zu übergeben. 

``mkdir -p`` um Unterverzeichnisse zu erstellen

### Übung (admin)

Gegeben ist:

``USER="admin"``

Gib folgende Strings mit echo korrekt aus:
- admin_backup
- admin_2026

Erkläre, warum folgende Zeile nicht das gewünschte Ergebnis liefert:

``echo "$USER_backup"``


```sh

USER="admin"

echo "${USER}_backup"
echo "${USER}_2026"

# echo "$USER_backup"

```

``echo "$USER_backup"`` ist falsh weil der _ auch ein teil eines Variablennamen sein kann und deshalb auf eine variable zugegriffen wird die es nicht gibt. Daher muss man die Variable mit von anderen Strings mit ``{}`` trennen 


### Übung (log file names)

Gegeben ist:

``DIR="/var/log/"``

Erzeuge mit echo folgende Ausgaben unter Verwendung der Variabelen DIR:

- /var/log/nginx
- /var/log/apache


```sh

DIR="/var/log/"

echo "${DIR}nginx"
echo "${DIR}apache2"

```
Hier muss man ebenfalls die geschwungenen Klammern verwenden um den Variablennamen vom String zu trennen. 

### Übung (backup)

``BASE="backup"``

``DATE="2026-01-16"``
